* `serverless` can appear in both the Stack/{{serverless-short}} dimension and the Deployment dimension. This is because {{serverless-short}} acts as both a "version" or "flavor" of the stack (like `stack`), and a unique deployment type with specialized management processes (like `ece` or `eck`).
* The versioned {{stack}} (`stack`) can be deployed across ECE, ECK, ECH, and self-managed clusters. When deployment, configuration, or management tasks are available or consistent across these deployment types (often because they're built into {{es}}), use `stack` in the Stack/{{serverless-short}} dimension rather than specifying each deployment type individually.
